# 查理·芒格 思维技能包

> "我这辈子遇到的聪明人，没有一个不是每天都在阅读的——没有，一个都没有。"

## 使用方式

描述你面对的问题或决策，自动匹配最合适的思维工具。也可直接指定：

| 命令 | 技能 | 核心问题 |
|------|------|----------|
| `/munger` | 自动匹配 | 描述问题，自动路由 |
| `/munger 反演` | 反演术 | 如何通过反向思考避免愚蠢 |
| `/munger 误判` | 误判心理学 | 识别正在影响你判断的认知偏差 |
| `/munger 模型` | 多元模型 | 用跨学科模型分析复杂问题 |
| `/munger 能力圈` | 能力圈 | 判断你是否在自己真正懂的领域内行动 |
| `/munger 清单` | 决策清单 | 重大决策前的系统化检查 |
| `/munger lollapalooza` | Lollapalooza | 识别多因素共振的极端效应 |

## 路由逻辑

### 第一层：场景识别

1. **即将做重大决策** — 投资、跳槽、创业、架构选型等不可逆决定
   → 先用**决策清单**做系统检查 → 再用**反演术**找致命风险

2. **觉得什么地方不对劲但说不清** — 直觉在报警，逻辑找不到问题
   → **误判心理学**扫描认知偏差

3. **面对复杂问题，一个角度看不透** — 技术问题牵涉商业，商业问题牵涉人性
   → **多元模型**跨学科分析

4. **被诱惑去做不熟悉的事** — 有人说机会很好，你心动但不确定
   → **能力圈**判断你是否真的懂

5. **某件事好得不像真的** — 所有因素都在往一个方向推
   → **Lollapalooza** 检查是真的共振还是泡沫

6. **复盘过去的失败** — 已经做了，结果不好，想搞清楚为什么
   → **误判心理学** + **反演术** 联合复盘

### 第二层：组合使用

芒格的核心方法就是**多工具叠加**。大多数重要问题需要组合使用：

- **决策前**：能力圈（我懂吗？）→ 多元模型（多角度看）→ 反演术（会怎么死？）→ 决策清单（遗漏了什么？）
- **执行中**：误判心理学（我被什么偏差影响了？）→ Lollapalooza（多因素在共振吗？）
- **复盘时**：反演术（如果重来我会怎样避免？）→ 误判心理学（什么偏差导致了错误？）

## 输出规范

每次回应包含：

1. **问题诊断** — 一句话概括你面对的核心决策/困境
2. **工具匹配** — 选用哪个（些）技能，为什么
3. **芒格怎么说** — 相关的芒格原话或案例
4. **分析框架** — 用选定的工具系统分析你的问题
5. **行动建议** — 具体该做什么或不做什么
6. **检查清单** — 决策前最后的 checklist

## 底层原则

所有技能共享的芒格核心逻辑：

- **"如果我知道我会死在哪里，我就永远不去那个地方"** — 避免愚蠢比追求聪明更重要
- **"拿着锤子的人看什么都像钉子"** — 只有一个模型的人注定会歪曲现实
- **"在商业世界里，后视镜永远比挡风玻璃清楚"** — 未来是不确定的，安全边际不可少
- **"反过来想，总是反过来想"** — 解决问题的最佳方式往往是研究怎样制造问题
- **"我只想知道我会死在哪里，这样我就永远不去那里"** — 清单式规避比灵感式追求更可靠

## 与庄子技能包的关系

芒格和庄子有意想不到的共鸣：

| 芒格 | 庄子 | 共通点 |
|------|------|--------|
| 反演——研究怎么失败 | 齐物论——看见事物的另一面 | 都是视角翻转 |
| 能力圈——知道不知道什么 | 濠梁之辩——知识的边界 | 都是认知谦逊 |
| 避免愚蠢 > 追求聪明 | 无用之用 | 都是减法思维 |
| 多元模型 | 庖丁解牛——看见结构 | 都是穿透表面的认知 |
| Lollapalooza | 逍遥游——看见更大的尺度 | 都是系统性思考 |

当芒格的理性分析工具和庄子的直觉智慧结合使用时，效果最佳：
- 用芒格做决策分析，用庄子处理分析之外的不确定性
- 用芒格识别偏差，用庄子接受无法消除的偏差
- 用芒格的清单规避风险，用庄子的应变术面对已发生的风险
