# 反演术 — 反过来想，总是反过来想

> "Invert, always invert." — 雅各比（芒格最常引用的数学家）

## 核心理念

大多数人思考问题的方式是"我怎么才能成功"。芒格说：**先想清楚怎么才能失败，然后避免那些事。**

"如果我知道我会死在哪里，我就永远不去那个地方。"

这不是悲观主义。这是工程师思维——**找到系统的故障模式，比设计完美系统更重要。**

为什么反演比正向思考更有效？
- 成功的路径有无数条，每条都不确定
- 失败的模式往往是有限的、可枚举的、重复出现的
- 避免确定的错误，比追求不确定的成功，性价比高得多

## 情境触发

当你遇到以下场景时，使用此技能：

- 即将做一个重大决策，需要评估风险
- 在规划一个项目/产品/策略
- 想搞清楚"这件事最可能怎么搞砸"
- 正在评估一个看起来很好的机会
- 复盘一次失败，想找到根本原因
- 感觉"什么都对"但隐约不安

## 行动步骤

### 第一步：定义"死法"清单

不要问"怎么做成"，先问"怎么做死"：

**操作：**
1. 写下你的目标（"做成X"）
2. 翻转它：**"如果我想确保X彻底失败，我该怎么做？"**
3. 认真列出所有能导致失败的行为和条件——至少列 10 条
4. 你会发现：很多"死法"你正在做或即将做

**举例——如果你想确保一个创业公司失败：**
- 花光现金还没找到 PMF
- 创始人之间对方向有根本分歧但不沟通
- 招了太多人但收入跟不上
- 忽视用户反馈，坚信自己的愿景
- 在不懂的领域做重大押注
- 没有竞争壁垒，纯靠执行速度
- ……

现在看看这个清单——你的公司正在犯其中几条？

### 第二步：做"验尸报告"（Pre-Mortem）

这是反演术最强的应用形式：

**操作：**
1. 假设你的项目/决策已经失败了（不是"可能失败"，是"已经失败了"）
2. 从未来回看：**"它是怎么死的？"**
3. 让所有相关人各自写下"死亡原因"——不讨论、不互相影响
4. 汇总所有死因，按"可能性 × 严重性"排序
5. 对排名前三的死因，制定具体预防措施

**为什么 Pre-Mortem 比"风险评估"有效：**
- 风险评估时人倾向于乐观（确认偏误——我想做这件事，所以我会低估风险）
- Pre-Mortem 把心态从"希望成功"切换到"已经失败"——绕过了乐观偏差
- 说"这个项目可能有风险"很难让人行动，说"这个项目已经死了，是因为X"会立即引起重视

### 第三步：反转你的假设

每个决策都建立在一组假设上。反演术要求你逐一翻转：

**操作：**
1. 列出你的关键假设（通常 3-5 个）
2. 对每个假设问："**如果这条是错的呢？**"
3. 评估：如果这条假设错了，整个决策还成立吗？
4. 找到"杀手假设"——一旦错了整个决策就崩塌的那条

**举例——"我们应该开发 App"的假设反转：**
- 假设1：用户需要这个功能 → 如果用户不需要呢？验证了吗？
- 假设2：我们能在 3 个月内开发完 → 如果需要 9 个月呢？还做吗？
- 假设3：市场窗口还在 → 如果竞品下月上线呢？
- 假设4：用户会从网页迁移到 App → 如果用户觉得网页够用呢？

找到杀手假设后，**在行动之前先验证这条假设**。

### 第四步：研究"别人的死法"

芒格大量阅读商业失败案例。你也应该：

**操作：**
1. 找到和你做的事类似的失败案例（不是成功案例！）
2. 问："他们是怎么死的？"
3. 对照自己："我有没有在犯同样的错？"
4. 特别关注那些看起来做对了所有事但仍然失败的案例——通常是因为他们忽视了某个结构性风险

**芒格的原话：**
"我总是试图通过弄清楚什么是愚蠢的，然后避免它，来把生活过好。我不去弄清楚什么是聪明的。"

### 第五步：对"好消息"反演

当所有信息都在说"做！"的时候，最需要反演：

**操作：**
- "这个投资/机会太好了" → **"如果这不是看起来那么好呢？我看到的好是因为什么被遮蔽了？"**
- "所有人都说应该做" → **"在什么条件下所有人都会是错的？"**（通常是泡沫或群体迷思）
- "数据完美支持这个结论" → **"什么样的数据能推翻这个结论？这样的数据可能存在吗？"**

芒格说："每当你看到一个特别好的结果，想想背后有没有幸存者偏差。你看到的可能只是运气好的那些。"

## 示例

**场景**：你要决定是否接受一个大客户的定制化需求——营收很大但会占用核心团队 60% 的精力。

**反演做法**：
1. 死法清单："如果我想让这次合作彻底搞砸我该怎么做？"——过度承诺交付时间、让核心产品团队去做定制开发导致主产品停滞、对客户的需求变更不设边界、没有替代方案当客户突然退出时
2. Pre-Mortem："6 个月后这个合作已经失败了，最可能的死因是什么？"——客户不断追加需求，团队精力耗尽，主产品错过市场窗口，客户最终还是不满意因为定制永远做不完
3. 反转假设：客户会长期合作吗？（如果只做 6 个月呢？）60% 精力够吗？（如果需要 90% 呢？）这个营收能持续吗？（如果客户明年换供应商呢？）
4. 别人的死法：有多少小公司因为被一个大客户绑架而失去了自己的产品路线？
5. 对好消息反演："营收很大"——**大到让我不敢理性分析了？**这本身就是危险信号

**决策：** 可能的做法是接受但设严格边界（固定范围、专门团队、不动核心产品人员），或者直接拒绝。无论哪种，反演帮你看到了正面分析看不到的风险。

## 反模式

| 错误 | 为什么错 |
|------|----------|
| 只做反演不做正面分析 | 反演是补充不是替代。你仍然需要正面评估机会和价值 |
| 用反演吓唬自己不敢行动 | 反演的目的是识别可管理的风险，不是证明一切都很危险 |
| Pre-Mortem 变成一言堂 | 如果只有一个人在想死法，你会遗漏。让所有人独立思考然后汇总 |
| 反演完了不采取行动 | 找到了风险但不做预防措施=白做 |
| 对小决策过度反演 | 不是所有事都需要反演。午饭吃什么不需要 Pre-Mortem |

## 退出条件

- 你面对的不是决策问题而是执行问题——已经决定了，需要做 → 切换到**决策清单**做执行检查
- 你已经反演够了但还是不敢行动 → 这不是分析问题是勇气问题
- 风险已经无法再降低——剩下的是不可消除的不确定性 → 接受它，做决定
- 你在用反演拖延决策 → 设一个截止时间，到了就必须决定
