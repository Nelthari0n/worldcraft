# 认知术 — 超越二元对立的思维方式

> 《齐物论》+《秋水》——你以为的真相，只是你站的位置。

## 原典

### 庄周梦蝶

庄周梦见自己变成蝴蝶，栩栩然蝴蝶也，自喻适志，不知周也。俄然觉，则蘧蘧然周也。不知周之梦为蝴蝶与，蝴蝶之梦为周与？

——你确定你认为的"现实"就是现实吗？**确定性本身就是一种幻觉。**

### 朝三暮四

狙公赋芧，曰："朝三而暮四。"众狙皆怒。曰："然则朝四而暮三。"众狙皆悦。名实未亏而喜怒为用。

——总数没变，只是换了个说法。**人对信息的反应取决于框架，不取决于事实本身。**

### 秋水·河伯与海

秋水时至，百川灌河。河伯欣然自喜，以天下之美为尽在己。顺流而东行，至于北海，望洋而叹。

北海若曰："井蛙不可以语于海者，拘于虚也；夏虫不可以语于冰者，笃于时也。"

——河伯以为自己够大了。见到海才知道自己的渺小。**你在自己的系统里看到的"全部"，可能只是更大系统的一个角落。**

### 濠梁之辩

庄子与惠子游于濠梁之上。庄子曰："鯈鱼出游从容，是鱼之乐也。"惠子曰："子非鱼，安知鱼之乐？"庄子曰："子非我，安知我不知鱼之乐？"

——**知识的边界在哪里？你怎么知道你知道的是对的？你怎么知道你不知道的就一定不知道？**

## 情境触发

当你遇到以下状态时，使用此技能：

- 两方各执一词，你必须选边但双方都有道理
- 陷入"对错"之争，越争越僵
- 做决策时信息矛盾，无法判断哪个是真相
- 团队内对同一件事有完全不同的解读
- 觉得"显然应该这样"但别人显然不同意
- 信息过载，越分析越糊涂

## 行动步骤

### 第一步：暂停判断——标记你的"确定"

当你觉得"这明明就是对的/错的"时，标记这个确定感：

- "我确定X是对的"→ 追问："我是基于什么确定的？"
- 通常追问两三层就会到底：你的确定来自经验、权威、直觉、或群体共识
- 这些来源不一定错，但它们不是真理——它们是你的视角

**朝三暮四检验法：**
- 把同一件事用不同的方式描述
- 如果换个说法你的判断就不同了——那你判断的不是事实，是叙述

例如：
- "他拒绝了我的方案" vs "他对方案提出了不同的优先级考量"
- "这个项目失败了" vs "这个项目产出了有价值的经验和数据"
- 事实没变，但你的反应完全不同——这就是朝三暮四

### 第二步：找到对方的"河"

河伯以为自己是全部，直到看见海。当你和别人意见相左：

- **不要问"谁对"，问"我们各自站在哪里看"**
- 每个人的观点都是从自己的"河"里看出去的
- 你需要弄清楚的不是谁错了，而是大家分别看到了什么

**操作：**
1. 让每一方描述他们看到的"事实"（不是观点，是他们亲眼所见）
2. 找出差异：大家看到的事实为什么不同？通常是因为站的位置不同
3. 问："有没有一个更大的视角，能同时包含所有人看到的？"

### 第三步：齐物——不是"都对"，是"在各自的框架内都成立"

齐物论不是和稀泥的"你也对他也对"。它是：

- A 在 A 的框架内是对的
- B 在 B 的框架内也是对的
- **真正的问题不是 A 和 B 谁对，而是我们应该用哪个框架**

**操作：**
1. 把争论从"内容层"提升到"框架层"
2. 不说"你的结论错了"，而说"你用的框架适合X场景，但我们现在面对的是Y场景"
3. 找到适合当前场景的框架，在那个框架下做决策

例如：技术方案争论
- A 说用微服务（从可扩展性框架看）
- B 说用单体（从交付速度框架看）
- 真正要讨论的不是微服务vs单体，而是**当前阶段我们优先扩展性还是速度？**

### 第四步：濠梁之辩——知道你不知道什么

在做判断之前，区分三种状态：

1. **我知道的**——有直接证据或可靠来源
2. **我推断的**——基于已知信息的延伸，可能对也可能错
3. **我不知道的**——没有信息，任何判断都是猜测

**操作：**
- 对你即将做出的每个重要判断，标注它属于哪一类
- 对第1类：可以有信心
- 对第2类：明确告诉别人"这是推断"
- 对第3类：承认不知道，不猜

"子非鱼安知鱼之乐"——如果你不确定，说"我不确定"比给一个看起来自信的错误答案好一万倍。

### 第五步：用"以明"做决策

庄子说"莫若以明"——不是用逻辑分析到底（分析到底会无穷后退），而是在承认不确定性之后，用直觉做一个"够好"的判断。

- 不追求"正确答案"——很多决策没有正确答案
- 追求"在当前信息下最合理的判断"
- 做完决策后保持开放：新信息出现时愿意修正

## 示例

**场景**：产品和技术团队就架构方案争论不休。产品说"我们需要快速迭代所以用低代码"，技术说"低代码没法定制会成为技术债"。你是负责人，必须做决定。

**认知术做法**：
1. 暂停判断：你本能觉得技术团队是对的——但检查一下，是因为你自己是技术背景吗？朝三暮四检验：把"低代码导致技术债"换成"低代码降低了验证成本"——是不是感觉不同了？
2. 找到各自的河：产品看到的是市场窗口（3个月后就晚了），技术看到的是6个月后的维护成本。两个都是真实的
3. 齐物：不是"低代码好不好"的问题，是"当前阶段时间成本和未来维护成本哪个优先"的问题
4. 标注不确定：市场窗口真的只有3个月吗？（推断）技术债真的那么严重吗？（推断）——对双方的推断都要问证据
5. 以明做决定：可能的方案是"核心模块自研，非核心模块用低代码"——不完美，但在当前信息下最合理

## 反模式

| 错误 | 为什么错 |
|------|----------|
| "你说的也对他说的也对"然后不做决定 | 齐物论不是和稀泥。承认多种框架成立是为了做更好的决定，不是为了逃避决定 |
| 无限分析——"我再想想" | 庄子明确反对无穷辩论。"以明"就是在某个点上停止分析，做判断 |
| 把"不确定性"当作借口不表态 | 承认不确定是诚实，但你的角色要求你做判断时，不确定也要判断 |
| 相对主义——"什么都是对的" | 齐物不是说没有标准，是说标准是情境性的。在具体情境中，有更好和更差的选择 |

## 退出条件

- 问题不是认知冲突而是利益冲突——大家看到的一样，只是想要的不同 → 这是谈判问题，不是认知问题
- 有明确的事实可以查证——不需要哲学，需要数据 → 去查证
- 时间不允许深入思考——需要立即行动 → 用直觉做"够好"的判断，事后再反思
- 你用齐物论来回避做出明确判断 → 你需要的是勇气，不是认知工具
